[Yeobaek 시스템 설계 노트]

작성일: 2025-11-17
작성자: 여백 Automation Lab

---

1. 전체 시스템 개요
---------------------------
여백 시스템은 크게 Front-End, Back-End, RAG Engine, Data Layer로 구성된다.

- Frontend: React 기반 Yeobaek HUB 웹
- Backend: FastAPI를 통해 챗봇 API 제공
- RAG Engine: Graph-RAG 기반 정보 검색 강화 구조
- Data Layer: 규정 문서, 학사정보, 도서관 정보, KDC 온톨로지, 여백 문서 등

---

2. RAG 구조
---------------------------
문서 → 청크 분할 → 임베딩 생성 → 로컬 벡터DB(Chroma) 저장

검색 흐름:
- 사용자가 질의
- 벡터DB에서 semantic search
- 상위 N개 문서 chunk 선정
- LLM에게 "문맥 + 질의" 형태로 전달

---

3. Graph-RAG 적용 구상
---------------------------
Graph-RAG는 단순 chunk retrieval이 아니라 문서 간 semantic 관계를 그래프로 모델링한다.


4. 향후 계획
---------------------------
- Firebase와 연동하여 사용자 대화 로그 저장
- KDC 온톨로지 → graph로 변환하여 실험
- Yeobaek-bot의 의사결정 흐름 명세서 작성
- HillMap, Automation, Recommender 기능 통합
